# Лабораторная работа: Тестирование методом "черного ящика"

## Цель работы
Освоить методы тестирования "черным ящиком", научиться составлять тест-кейсы и баг-репорты.

## Задачи
1. Провести автоматическое тестирование приложения
2. Провести ручное тестирование
3. Составить баг-репорты на найденные дефекты
4. Проанализировать результаты тестирования

## Теоретическая часть

### Метод черного ящика
Тестирование "черным ящиком" - метод тестирования ПО, при котором тестировщик не имеет доступа 
к исходному коду и внутренней структуре программы. Тесты основываются на:
- Спецификации требований
- Входных и выходных данных
- Поведении системы

### Типы тестовых случаев:
1. **Позитивные тесты** - проверка корректных данных
2. **Негативные тесты** - проверка некорректных данных
3. **Граничные значения** - проверка на границах диапазонов
4. **Эквивалентные классы** - группировка входных данных

### Шаг 2: Ручное тестирование
1. Запустите `buggy_calculator.py`
2. Проведите тестирование по следующему плану:

#### Тест-кейсы для проверки:
1. **Арифметические операции:**
   - 5 + 3 = 8
   - 10 - 15 = -5
   - 4 * 3 = 12
   - 20 / 4 = 5

2. **Граничные значения:**
   - 0 + 0

3. **Некорректный ввод:**

4. **Особые случаи:**
   - Деление на 0
